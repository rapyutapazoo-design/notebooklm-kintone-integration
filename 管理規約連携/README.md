# 管理規約アプリ NotebookLM 連携

kintone の「管理規約アプリ」で、レコードに登録した NotebookLM ノートの URL を
ワンクリックで開けるカスタマイズです。

- **一覧画面（PC / モバイル共通）**: 「規約タイトル ＋ NotebookLMボタン」を
  **カードスタイル**で全件表示します。
- **詳細画面（PC / モバイル）**: ノートを開くボタンを表示します。
- レコードごとに異なるノートへ遷移します（URL はレコード単位で手動登録）。
- URL 未登録のレコードはボタンが「未連携」とグレー表示されます。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `notebook-link.js` | カード一覧 + 詳細ボタンの表示ロジック（PC / モバイル） |
| `notebook-link.css` | カード・ボタンのスタイル（アイコンはインライン SVG） |
| `README.md` | 本手順書 |

## 1. 事前準備: フィールドの確認・作成

対象アプリ（管理規約アプリ）に、以下の2フィールドが必要です。

| 用途 | パーツ | フィールドコード | 備考 |
|---|---|---|---|
| ノートの URL | 「リンク」（種類: **URL**） | `notebook_url` | 綴り厳守。違うと動きません |
| 規約タイトル | 文字列1行 など | `title` | カードの見出しに使用 |

- フィールドコードの設定場所: フォーム編集画面 → 各フィールドの歯車アイコン → フィールドコード
- フィールドコードを変えたい場合は、`notebook-link.js` 冒頭の `CONFIG.fieldCode` /
  `CONFIG.titleFieldCode` を合わせて変更してください。

## 2. 一覧ビューの作成（カスタマイズビュー）

カードUIは、kintone 公式の「カスタマイズビュー」に描画します。

1. 「アプリの設定」→「一覧」→ **新しい一覧を追加**（または既存一覧を編集）
2. **表示形式** を **「カスタマイズ」** にする
3. **HTML** 欄に、以下の1行だけを記述する:

   ```html
   <div id="notebook-card-list"></div>
   ```

4. 「保存」する

> - この一覧は **PC・モバイル共通** で使われます（同じビューを両方で表示）。
> - 要素の id を変えたい場合は `notebook-link.js` の `CONFIG.cardContainerId` を合わせてください。

## 3. ファイルの適用（JS / CSS アップロード）

1. 「アプリの設定」→「設定」タブ →「JavaScript / CSS でカスタマイズ」を開く
2. **【PC 用】**
   - 「JavaScript ファイル」に `notebook-link.js` をアップロード
   - 「CSS ファイル」に `notebook-link.css` をアップロード
3. **【スマートフォン用】**
   - 同じ `notebook-link.js` をアップロード
   - 同じ `notebook-link.css` をアップロード
4. 「保存」をクリックし、アプリを **「アプリを更新」** する。

## 4. 運用: レコードにURLを登録

1. レコードを開く（または新規作成）
2. `title` に規約タイトル、`notebook_url` に NotebookLM ノートの URL
   （`https://notebooklm.google.com/...`）を入力して保存

## 5. 動作確認チェックリスト

- [ ] `notebook_url`（リンク/URL）と `title` のフィールドコードが正しい
- [ ] 一覧ビューの表示形式が「カスタマイズ」で、HTML に `<div id="notebook-card-list"></div>` がある
- [ ] **PC 一覧**: 「タイトル＋ボタン」のカードが全件並ぶ
- [ ] **モバイル一覧**: PC と同じカードが並ぶ
- [ ] **PC 詳細**: 画面上部メニュー右にボタンが表示される
- [ ] **モバイル詳細**: フィールド直下にボタンが表示される
- [ ] **未登録レコード**: カード内ボタンが「未連携」グレーになる
- [ ] ボタンをタップすると新しいタブで対象ノートが開く

## 設定（CONFIG）

`notebook-link.js` 冒頭の `CONFIG` で挙動を調整できます。

| キー | 既定値 | 説明 |
|---|---|---|
| `fieldCode` | `notebook_url` | URL を格納するリンクフィールドのフィールドコード |
| `titleFieldCode` | `title` | カード見出しに使うフィールドコード |
| `label` | `NotebookLMで開く` | ボタンの文言 |
| `inactiveLabel` | `未連携` | URL 未登録時の文言 |
| `noTitleLabel` | `(タイトル未設定)` | タイトル未入力時の文言 |
| `allowedScheme` | `https://` | 許可する URL スキーム（`javascript:` 等は無効化） |
| `cardContainerId` | `notebook-card-list` | カスタマイズビューに置く要素の id |

## 補足・既知の制約

- セキュリティのため、`https://` で始まらない URL はリンクとして無効化されます。
- カード一覧は**カスタマイズビュー前提**です。万一カスタマイズビューが未設定の環境では、
  フォールバックとして一覧ヘッダー直下にカードを描画しますが、標準の表が併せて表示される
  ため、上記「2. 一覧ビューの作成」の設定を推奨します。
- 件数は10件以下を想定し、全件を一度に描画します（ページングは行いません）。
- アイコンは外部画像を使わずインライン SVG で描画しているため、追加の画像アップロードは不要です。
